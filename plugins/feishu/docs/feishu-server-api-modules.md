# 飞书开放平台服务端 API（自建应用）功能清单

本文用于快速了解飞书开放平台服务端 API 在**自建应用**场景下可用的业务能力（模块级），便于在 Node.js 服务端集成时进行模块选型。

> 说明
> - 官方完整列表入口：<https://open.feishu.cn/document/server-docs/api-call-guide/server-api-list>
> - SDK 语义化调用通常采用 `client.<module>.<resource>.<method>` 形式（例如：`client.im.message.create`）。
> - 飞书 API 迭代较快，新增/下线能力请始终以官方文档页面为准。
> - 该页面展示为动态内容；本文采用“模块/能力域”口径汇总自建应用可用功能。

## 7. 事件订阅能力（服务端常用）

以下能力常与上述模块联动使用：
- `EventDispatcher`：Webhook 事件分发与处理
- `WSClient`：长连接模式接收事件
- `CardActionHandler`：消息卡片交互回调处理

典型流程：
1. 通过事件订阅接收业务事件（如 `im.message.receive_v1`）
2. 在回调中调用对应模块 API（如 `client.im.message.create`）
3. 返回处理结果或更新卡片内容

## 8. Node.js 服务端接入建议

- 按业务优先级拆分模块接入，避免一次性申请过多权限（可参考 [优先实现的功能](./priority-features.md)）。
- 优先基于 `client.<module>.<resource>.<method>` 的语义化接口开发。
- 对于历史或未封装语义方法的接口，使用 `client.request(...)` 兜底。
- 所有字段、权限范围、频控与错误码以官方 API 文档为准。
